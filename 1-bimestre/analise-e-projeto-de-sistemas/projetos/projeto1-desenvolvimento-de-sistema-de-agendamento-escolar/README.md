# 📘 PROJETO 1: Desenvolvimento de Sistema de Agendamento Escolar

# 🧩 PROBLEMA

A escola enfrenta um problema recorrente:

Os alunos precisam falar com professores fora do horário de aula, mas:

- Não sabem quando o professor está disponível.
- Vão até a sala dos professores e não encontram ninguém.
- Vários alunos tentam falar com o mesmo professor ao mesmo tempo.
- Não existe controle de ordem de atendimento.

A direção deseja um **Sistema de Agendamento de Atendimento aos Professores**, acessível aos alunos e professores.

A equipe de vocês foi contratada para **analisar e projetar esse sistema**.

# 📋 RESOLUÇÃO

Vocês devem propor uma solução que atenda aos seguintes pontos mínimos:

### O sistema deve permitir:

- Selecionar um professor.
- Escolher data e horário.
- Registrar o agendamento.
- Evitar conflito de horários.
- Permitir visualização da agenda pelo professor.

Além disso, o sistema deve:

- Ser simples e intuitivo.
- Funcionar em celular.
- Ser rápido e organizado.

# 🧠 DESENVOLVIMENTO DO PROJETO

# 🔹 ETAPA 1 — Levantamento de Requisitos

Cada dupla deve produzir:

### ✅ 5 Requisitos Funcionais (mínimo)

Exemplo de estrutura esperada:

```
RF01 - O sistema deve permitir que o aluno selecione um professor.
```

### ✅ 3 Requisitos Não Funcionais (mínimo)

Exemplo de estrutura esperada:

```
RNF01 - O sistema deve funcionar em dispositivos móveis.
```

# 🔹 ETAPA 2 — Diagrama de Casos de Uso

A dupla deve transformar os requisitos em ações do usuário.

Vocês devem:

- Definir atores
- Criar pelo menos 4 casos de uso
- Desenhar o diagrama (no PDF)

# 🔹 ETAPA 3 — Protótipo de Baixa Fidelidade

A dupla deve transformar os requisitos e casos de uso em telas.

Devem ser desenhadas:

- Tela do aluno
- Tela do professor
- Garantir que todos os requisitos apareçam nas telas

# 📦 Entrega

O PDF deve conter:

- Capa (nome da escola, logo da escola, título do projeto, disciplina, nome da dupla e nome do professor)
- Nome do arquivo deve ser exatamente: “projeto-agendamento-escolar.pdf”
- Fonte: 12pt, Arial
- Espaçamento: padrão
- 1 folha por etapa
- Boa organização do projeto
- Deve ser enviada a pasta de “Projetos” no Codespace do Github (a dupla deve ter em cada um de seus repositórios)

Boas práticas! 🤙
