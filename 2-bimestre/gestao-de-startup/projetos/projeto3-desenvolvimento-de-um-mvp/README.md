# 📘 PROJETO 3: DESENVOLVIMENTO DE UM MVP

# 🚀 PROBLEMA

A escola percebeu que vários problemas do dia a dia continuam sem solução simples.

Alguns exemplos:

- dificuldade de organização dos alunos
- comunicação ruim entre turmas
- atraso em atividades
- dificuldade de divulgar eventos
- desperdício de materiais
- dificuldade em encontrar informações

Sua equipe deverá:

> Identificar um problema real da escola ou comunidade e desenvolver uma solução simples, prática e validável.
> 

⚠️ Importante:

A solução deve ser algo possível de existir na vida real.

# 🧩 PROJETO

Cada equipe deverá criar:

✅ Identificação do problema

✅ Explicação do público afetado

✅ Solução proposta

✅ MVP da solução

✅ Teste rápido com pessoas reais

✅ Melhorias após feedback

# 👥 Formação das equipes

- Grupos de 4 a 5 alunos

# 🔄 ESTRUTURA DO PROJETO

# ⏱️ ETAPA 1 – Identificação do problema

O grupo deve discutir problemas reais observados:

📌 Na escola

📌 Na comunidade

📌 No cotidiano dos estudantes

## ⚠️ Regra importante

O problema precisa:

- existir de verdade
- afetar pessoas reais
- ser observável

## 📌 Exemplos aceitáveis

- alunos esquecem atividades
- dificuldade para saber horários
- comunicação ruim entre grupos
- dificuldade para divulgar trabalhos
- desperdício de papel
- etc.

# ⏱️ ETAPA 2 – Pensamento crítico e análise

Agora o grupo deve responder:

```
Qual é o problema real?

O que causa esse problema?

Quem é afetado?

Como isso poderia ser melhorado?
```

📌 Objetivo:

Não criar solução sem entender o problema.

# ⏱️ ETAPA 3 – Desenvolvimento da solução

Agora o grupo cria a solução.

A solução pode ser:

- serviço
- organização
- aplicativo
- sistema simples
- formulário
- quadro organizacional
- método de comunicação

## ⚠️ Regra

A solução precisa ser:

- simples
- viável
- prática

# ⏱️ ETAPA 4 – Construção do MVP

Agora o grupo cria o MVP.

## 📌 O MVP pode ser:

### ✏️ Papel

- desenho de telas
- fluxo de funcionamento
- modelo da solução

### 📱 Tablet

- apresentação simples
- Canva
- slides
- formulário simples

## ⚠️ Regra importante

O MVP deve mostrar:

- como a solução funcionaria
- da forma mais simples possível

👉 Não precisa estar completo.

# ⏱️ ETAPA 5 – Validação real

Agora o grupo deverá mostrar sua ideia para:

- professores
    
    OU
    
- outros alunos (exceto da sua própria sala)

E perguntar:

```
Você usaria isso?

Isso realmente ajuda?

O que ficou confuso?

O que melhoraria?
```

📌 Isso simula validação de mercado.

# ⏱️ ETAPA 6 – Ajustes e melhoria

Com base no feedback:

- melhorar o MVP
- simplificar
- ajustar problemas encontrados

👉 Aplicando o ciclo:

```
IDEIA → TESTE → FEEDBACK → MELHORIA
```

# 📝 AVALIAÇÃO

A avaliação do projeto ocorrerá de acordo com:

- Participação
- Organização da equipe
- Clareza do problema
- Qualidade do MVP
- Aplicação do feedback
- Capacidade de resolver problemas
