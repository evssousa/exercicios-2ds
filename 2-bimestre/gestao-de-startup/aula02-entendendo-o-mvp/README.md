# 📘 AULA 22: ENTENDENDO O MVP: PRODUTO MÍNIMO VIÁVEL

## 🎯 Objetivos da aula

- Entender o conceito de **MVP (Produto Mínimo Viável)**
- Compreender a importância de lançar versões simples
- Diferenciar MVP de produto final
- Aplicar o conceito em ideias simples

## 🚀 O que é MVP?

MVP significa:

> **Produto Mínimo Viável**
> 

É a **versão mais simples possível de uma ideia**, que já pode ser testada.

## 🧠 Por que o MVP é importante?

Porque evita:

- perder tempo com algo complexo
- desenvolver algo que ninguém quer
- gastar recursos desnecessários

👉 Em startups:

> Melhor algo simples funcionando do que algo perfeito que nunca é testado.
> 

## 📌 Exemplo simples

### Ideia:

Criar um app de organização de tarefas

### Produto completo:

- login
- notificações
- categorias
- design completo

### MVP:

- apenas uma lista de tarefas funcionando

👉 Simples, mas já testável.

## ⚖️ MVP x Produto completo

```
MVP                    PRODUTO FINAL
Simples                Completo
Rápido                 Demorado
Testável               Refinado
```

## ⚠️ Erros comuns com MVP

- querer incluir muitas funções
- tentar deixar perfeito
- não testar com usuários

## 🧠 Regra importante

> Se está demorando muito, não é MVP.
> 

## 📌 Como pensar um MVP?

Se Pergunte:

- Qual é o problema principal?
- Qual é a solução mais simples possível?

Fim da aula!
