# 📘 PROJETO 4: ESTUDO DE VIABILIDADE, RISCOS E DOCUMENTAÇÃO DE SISTEMA

# 🧩 PROBLEMA

Uma clínica médica da cidade deseja modernizar seus processos e contratou sua equipe para realizar uma análise inicial.

Atualmente:

- Agendamentos são feitos por telefone.
- Horários são anotados em papel.
- Pacientes esquecem consultas.
- Há conflitos de horários.
- Não existe histórico digital dos atendimentos.

A clínica deseja um sistema capaz de:

- Agendar consultas.
- Consultar horários disponíveis.
- Cadastrar pacientes.
- Controlar médicos e especialidades.
- Consultar histórico de atendimentos.

Antes de investir dinheiro no desenvolvimento, a direção quer saber:

- O projeto é viável?
- Quais são os riscos?
- Quais requisitos o sistema deve possuir?

Sua equipe será responsável por elaborar essa análise.

# 📋 PROJETO

O sistema deverá atender minimamente aos seguintes requisitos:

### Requisitos Funcionais

```
RF01 - Permitir cadastro de pacientes.
RF02 - Permitir cadastro de médicos.
RF03 - Permitir agendamento de consultas.
RF04 - Permitir consulta de horários disponíveis.
RF05 - Permitir consulta do histórico de atendimentos.
```

### Requisitos Não Funcionais

```
RNF01 - O sistema deve ser fácil de utilizar.
RNF02 - O sistema deve proteger os dados dos pacientes.
RNF03 - O sistema deve funcionar em computadores da clínica.
```

# 🧠 DESENVOLVIMENTO

## 🔹 ETAPA 1 — Estudo de Viabilidade

Você deverá analisar se o projeto é viável.

```
PROJETO
    ↓
VIABILIDADE
    ↓
APROVADO OU NÃO
```

É preciso ter na documentação os seguintes estudos de viabilidade:

- Técnica
- Econômica
- Legal
- Operacional
- Cronograma

### Exemplos

### Técnica

```
A equipe possui conhecimento básico para desenvolver o sistema.
```

### Econômica

```
A clínica possui orçamento para o projeto.
```

### Legal

```
Será necessário proteger dados dos pacientes conforme a LGPD.
```

e etc.

## 🔹 ETAPA 2 — Análise de Riscos

Agora você deve identificar possíveis problemas.

```
RISCO
 ↓
IMPACTO
 ↓
PREVENÇÃO
```

Identifique pelo menos:

- 3 riscos
- impacto de cada risco
- estratégia preventiva

### Exemplos

### Risco

```
Perda de dados dos pacientes
```

### Impacto

```
Históricos médicos podem ser perdidos.
```

### Prevenção

```
Realizar backups periódicos.
```

### Outros exemplos

### Risco

```
Sistema ficar indisponível.
```

### Impacto

```
Consultas não podem ser agendadas.
```

### Prevenção

```
Manutenção e monitoramento do servidor.
```

Obviamente **NÃO COPIAR DOS EXEMPLOS**!

## 🔹 ETAPA 3 — Produção da Documentação

Agora você irá produzir os principais artefatos.

### 📄 Parte A — Mini SRS

Criar um documento contendo:

#### Objetivo do Sistema

Exemplo:

```
Controlar consultas e pacientes da clínica.
```

#### Requisitos Funcionais

Mínimo:

```
RF01
RF02
RF03
RF04
RF05
```

#### Requisitos Não Funcionais

Mínimo:

```
RNF01
RNF02
RNF03
```

### 📘 Parte B — Especificação de Caso de Uso

Criar um caso de uso.

#### Modelo

```
Caso de Uso: Agendar Consulta

Ator: Recepcionista

Fluxo Principal:
1. Selecionar paciente.
2. Selecionar médico.
3. Escolher horário.
4. Confirmar agendamento.
```

### 📚 Parte C — Glossário

Criar pelo menos 5 termos.

#### Exemplo

```
Paciente: Pessoa que realiza consultas.

Consulta: Atendimento médico agendado.

Especialidade: Área de atuação do médico.
```
